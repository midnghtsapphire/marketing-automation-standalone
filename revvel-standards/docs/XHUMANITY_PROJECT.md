## Purpose

Create the canonical project record for xHumanity and tie it to the repository's S2M / revvel-standards workflow.

## Mission

xHumanity is the standards-first project name to use when the work request asks for a project called **xHumanity**.

## Recommended Project Management Setup

Use either **GitHub Projects**, **Linear**, or both with the same project name:

- **GitHub Project title:** `xHumanity`
- **Linear project name:** `xHumanity`

## Minimum Workflow

The project board should at least track:

1. **Backlog** — raw requests and research leads
2. **Deep Research** — revvel-standard discovery, market research, and sourcing
3. **Ready to Build** — scoped work with required artifacts identified
4. **In Build** — implementation underway
5. **Validation** — testing, ship-readiness checks, and review
6. **Shipped** — deployed or ready for handoff

## Minimum Fields
